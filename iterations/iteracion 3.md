# Iteración 3

| Aspecto| Descripción |
|------|-------------|
| Drivers Seleccionados | - Realizar pagos a través de la pasarela de MercadoLibre.<br> - Confirmar transacciones y gestionar su verificación.  <br>   - Uso de protocolos HTTP/REST. <br> - Interoperabilidad: Integración con la pasarela de MercadoLibre mediante API REST.  <br> - Rendimiento: Procesar transacciones en menos de 2 segundos.  <br> - Seguridad: Proteger transacciones mediante encriptación y autenticación. <br> - Disponibilidad: Encolar solicitudes de pago en caso de fallo del servicio externo. |
| Objetivo de la Iteración |  Desacoplar el módulo de Pagos creando un microservicio que soporte el procesamiento de pagos. |
| Elemento a refinar |  Módulo de Pagos.|
| Elección de Conceptos de Diseño |  <ul><li> **Microservice** para desacoplar Pagos. </li><li> **API REST**: asegura un protocolo estándar para la integración. </li><li>  **API gateway**: punto de entrada centralizado para redirigir solicitudes, manejar politicas de seguridad, auntenticacion y balanceo de carga.   </li><li>  **Database per Service** para Pedidos.</li><li> **Circuit Breaker**: para manejar fallos de la pasarela.</li><li> **Message Queue**: para encolar pagos fallidos. </li><li>  **Encrypt Data**: para proteger los datos y asegurar que solo las usuarios autorizados puedan acceder a ellos.</li><li> **Authenticate Actors**: para verificar la identidad de un usuario.</li></ul>|
| Instanciar Elementos Arquitectónicos | Microservicio de Pagos: Responsable de procesar y confirmar transacciones. <br>  Interfaces: API gateway: como unico punto de entrada al sistema que redirige las solicitudes, API REST para integración con MercadoLibre. <br>  Cola de Mensajes: Encolará pagos no procesados debido a fallos.<br> Base de datos para el microservicio de pagos.|
| Diagramas y  Decisiones | Decisiones documentadas en **ADR 1**, **ADR 2**, **ADR 3**, **ADR4**, **ADR6** y los diagramas correspondientes a la iteracion 3 son: **it3_view_component**, **it3_diagrama_de_secuencia**. |
